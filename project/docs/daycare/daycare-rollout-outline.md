# Daycare Rollout Outline

This document belongs to the staged daycare side of the provider-program-platform package.

## Objective

Finish the daycare angle so the product is not just generic provider software.

## Required daycare pieces

- daycare site profile
- dismissal window setup
- classroom handoff workflow
- authorized pickup rules
- family intake packet
- future ride permission
- sports, appointment, and special-event transport readiness

## Why this matters

Daycare transportation fails when:

- release rules are unclear
- family packets are incomplete
- parents only enroll after a ride becomes urgent
- site-level dismissal workflow is never defined

## Staged result

The package should be able to support later rollout for:

- normal daycare dismissals
- one-off future rides
- football games and other sports
- appointments
- after-school and special programs
