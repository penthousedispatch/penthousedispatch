# Daycare Provider Guide

This guide explains how a daycare, school, or program should use the Penthouse platform with the dispatch team.

## 1. Sign up as a platform admin first

Before a daycare or program can manage children, subscriptions, or transportation, the provider needs to create a company account and sign up as an admin on the platform.

That admin account is the account that should:

- complete company onboarding
- manage the program profile
- maintain the child roster
- manage parent subscriptions
- control the parent incentive program

If the provider never signs up as an admin, the rest of the setup will be messy fast because dispatch will be working from incomplete data.

## 2. What the provider controls

The provider side is responsible for:

- enrolling families
- pushing all families to enroll early, not only when a ride is already needed
- keeping child rosters accurate
- setting pickup and dropoff expectations
- sending schedule updates early
- helping parents understand the subscription and incentive program

The provider should not treat the dispatch team like they already know the children. The platform only works well when the admin account keeps the roster and contact info current.

## 3. What to set up first

Before starting service, the provider should create:

- a program record
- a contact person for transportation issues
- the service address
- service days
- pickup windows
- release and safety rules

Then the provider should add each child with:

- child name
- guardian name
- guardian phone
- pickup address
- dropoff address
- pickup days
- mobility or ride notes

## 4. Enroll families before they need service

Do not wait for a transportation problem before asking parents to sign up.

The better model is:

- every family enrolls the child now
- the child stays dispatch-ready in the system
- rides can be created later when needed

This matters because some children:

- cannot use bus service
- may need one-off rides later
- may need transportation for doctor visits, therapy, programs, or sports
- may suddenly need dispatch because the normal transportation plan breaks

Examples of future rides:

- medical appointments
- after-school programs
- football games
- school events
- summer programs
- temporary service gaps when the bus is not available

If the family is already enrolled, dispatch can move fast.

If the family is not enrolled, the provider creates a last-minute scramble.

## 5. How to manage the roster

The roster is the source of truth.

Use it for:

- new enrollments
- future-use enrollments
- pauses
- waitlist riders
- guardian phone changes
- schedule changes
- ride-note changes

Good rule:

- if it changed in real life, update it in the platform the same day

## 6. How parent subscriptions should be explained

If your program uses platform subscriptions, explain:

- what the family is paying for
- that the subscription keeps the child ready for future dispatch, even before active rides begin
- whether the plan is monthly, annual, or trial
- when charges begin
- whether autopay is turned on
- what happens if the family pauses or cancels

Do not leave families guessing. Confusion around billing turns into dispatch problems fast.

## 7. Parent incentive program setup

The platform can offer a parent incentive program for subscribing.

Recommended structure:

- signup reward: account credit when a family joins
- retention rule: family stays active for a minimum number of months
- referral bonus: extra credit when the parent brings another family onto the platform

The goal is not just to reward current riders.

The goal is to get all parents enrolled now so the provider has a full dispatch-ready roster for future rides too.

What the provider should explain clearly:

- the reward amount
- how the reward is applied
- when the reward becomes valid
- whether referral codes are required

## 8. Day-to-day workflow

Each day the provider should:

1. Confirm who is riding.
2. Remove absences early.
3. Check for new family or address changes.
4. Send unusual ride notes to dispatch before pickup time.
5. Make sure release contacts are current.

If a child should not ride that day, update the platform or notify dispatch early. Late changes waste driver time and create avoidable billing arguments.

## 9. Working with dispatch

Dispatch needs clean information, not partial information.

When contacting dispatch, include:

- child name
- program name
- exact issue
- pickup or dropoff address if it changed
- callback number
- whether the issue is urgent for the current trip

Bad message:

- "Call me about pickup"

Better message:

- "Jordan Lee at Bright Start Daycare is absent today. Remove the 3:15 PM pickup. Guardian callback is 555-111-2222 if needed."

## 10. Release and safety rules

The provider must document:

- adult handoff requirements
- children who cannot be left unattended
- pickup authorization rules
- behavior or supervision notes
- mobility accommodations

If a child has strict release conditions, save that in the platform before the trip exists. Do not rely on memory.

## 11. Tracking and parent communication

When a trip is active, parents may receive a rider tracking link.

The provider should:

- tell parents when tracking is available
- explain that the link is for active trip visibility
- remind parents to keep their phone nearby on ride days

## 12. Quick provider checklist

- Sign up as a company admin on the platform first.
- Get all parents to enroll, even if they do not need rides right now.
- Keep the program profile current.
- Keep the child roster current.
- Update guardian and address changes fast.
- Explain subscriptions in simple terms.
- Explain that the platform is for current rides and future rides.
- Explain the parent incentive program before families join.
- Send dispatch exact facts, not vague messages.
