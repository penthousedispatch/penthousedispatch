# Provider Program Platform Compliance Outline

This document belongs to the staged provider-program-platform package only. It is not connected to the live product yet.

## Procurement

Prepare for later buyers such as:

- county agencies
- school districts
- nonprofit program operators
- managed care or NEMT-adjacent partners

Baseline staged controls:

- provider admin identity and onboarding
- contract status
- insurance verification status
- background check status
- training attestation status
- policy acknowledgment status

## Reporting

The staged package should be able to produce later:

- parent enrollment readiness reports
- future ride demand reports
- utilization reports by program and use case
- incident and guardian release exception reports
- incentive exposure and subscription reports

## Safety controls

The staged package should block or flag unsafe operations when:

- guardian release rules are missing
- emergency or incident escalation status is unresolved
- required training attestations are missing
- provider safety controls are incomplete

## Why this is staged now

If this side of the platform is ever sold into institutional or government-influenced workflows, procurement and compliance expectations will show up immediately. It is better to stage the structure now than bolt it on later under pressure.
